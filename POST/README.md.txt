# Power Query: «Отбор.csv» → листы BASE и LOAD (Excel 2016, русская локаль)

## Формулировка задачи и техническое задание

Требуется решение на **Power Query** для **Microsoft Excel 2016** (русский интерфейс).

1. Загрузить данные из файла **`Отбор.csv`**, расположенного **в той же папке**, что и книга Excel с запросами, на лист **`BASE`** (все строки и столбцы из CSV).
2. На листе **`LOAD`** сформировать таблицу:
   - 1-й столбец: **Короткое наименование ТБ**;
   - 2-й столбец: **ГОСБ**;
   - далее столбцы с заголовками типов отбора (в указанном порядке).
3. В строках — **уникальные сочетания** «ТБ + ГОСБ» из CSV.
4. В ячейке на пересечении строки (ТБ, ГОСБ) и столбца типа отбора вывести всех **КМ**, для которых одновременно:
   - в столбце **ВЫБРАН** значение **1** (допускается и текстовая «1»);
   - **Короткое наименование ТБ** и **ГОСБ** совпадают со строкой;
   - в поле **ОТБОР** **содержится** строка с названием типа отбора (сравнение **без учёта регистра**), в соответствии с заголовком столбца.
5. Формат вывода ФИО в одной ячейке:
   - исходно в CSV: полные строки вида `"Иванов Иван Иванович"`;
   - в ячейке: сокращённые **`Фамилия И.О.`** (инициалы из первых букв остальных слов), каждый следующий КМ — **с новой строки** (в Excel отображается как Alt+Enter).

**Типы отбора (заголовки столбцов после ГОСБ):**

- `ЗОЛОТО`
- `MAX BADGES`
- `СТРАЙК РОСТ ОД`
- `СТРАЙК ЭФФЕКТИВНОСТЬ`
- `ЭФФЕКТИВЕН`
- `РОСТ ОД`
- `>50% ТЕМП ОД`
- `>50% ТЕМП ОД ТБ`

**Ключевые поля CSV (ожидаемые имена столбцов после импорта):**  
`employeeNumber`, `КМ`, `Короткое наименование ТБ`, `ГОСБ`, `ОТБОР`, `ВЫБРАН`.

## Описание решения

В каталоге `src/PowerQuery/` хранятся **тексты запросов** на языке **M** (расширение `.pq` — удобно копировать в «Дополнительный редактор» Power Query в Excel).

| Файл запроса | Назначение |
|--------------|------------|
| `Настройка_ПутьККаталогу.pq` | Возвращает путь к папке книги через **именованную область** `КаталогРабочейКниги` в Excel. |
| `Отбор_На_BASE.pq` | Читает `Отбор.csv`, автоматически определяет разделитель (**`;`** или **`,`**), использует кодировку **Windows-1251**, поднимает заголовки. Результат загружается на лист **BASE**. |
| `LOAD_Сводная.pq` | Строит таблицу для листа **LOAD**: уникальные пары ТБ+ГОСБ, столбцы по типам отбора, фильтрация и склейка КМ с переносом строки `#(lf)`. |
| `Один_Запрос_BASE_LOAD.pq` | Единый самодостаточный запрос с подробными шагами: импорт CSV + построение LOAD. Переменная `РежимВывода` переключает итог (`BASE`/`LOAD`). |

**Почему именованная область для пути:** в Power Query нет универсальной встроенной функции «путь к текущей книге» во всех версиях; надёжный способ — одна ячейка с формулой каталога и именованная область на неё.

### Пошаговая настройка в Excel 2016

1. Сохраните книгу в ту папку, где будет лежать **`Отбор.csv`**.
2. В любой лист (можно служебный) вставьте в ячейку **A1** формулу каталога книги:
   - `=ЛЕВСИМВ(ЯЧЕЙКА("имяфайла";A1);ПОИСК("[";ЯЧЕЙКА("имяфайла";A1))-1)`
3. Выделите эту ячейку и задайте **имя** `КаталогРабочейКниги` (Формулы → Определённое имя).
4. **Вариант A (рекомендуется для простоты поддержки):** **Данные** → **Создать запрос** → **Из других источников** → **Пустой запрос** — создайте один запрос и вставьте код из файла:
   - `Один_Запрос_BASE_LOAD.pq`
   - в начале запроса установите `РежимВывода = "LOAD"` (или `"BASE"` при необходимости);
   - все этапы будут видны в списке «Примененные шаги», что упрощает диагностику ошибок.
5. **Вариант B (два отдельных результата одновременно):** создайте три запроса и вставьте код из файлов:
   - `Настройка_ПутьККаталогу.pq`
   - `Отбор_На_BASE.pq` (в строке `Папка = #"Настройка_ПутьККаталогу"` имя должно совпадать с именем первого запроса)
   - `LOAD_Сводная.pq` (ссылка ` #"Отбор_На_BASE"` — имя второго запроса)
6. Для варианта B: **`Отбор_На_BASE`** загрузите как таблицу на лист **`BASE`**.
7. Для варианта B: **`LOAD_Сводная`** загрузите как таблицу на лист **`LOAD`**.
8. При переносе файлов обновляйте данные: **Данные** → **Обновить все**.

Разделитель определяется автоматически (`;` или `,`). Если ваш CSV в UTF-8, замените `Encoding = 1251` на `Encoding = 65001`.

## Переменные и функции (в коде M)

### Запрос `Настройка_ПутьККаталогу`

| Имя | Смысл |
|-----|--------|
| `Источник` | Таблица из именованной области `КаталогРабочейКниги`. |
| `ИмяПервогоСтолбца` | Имя первого столбца таблицы (на случай отличия от `Column1`). |
| `ПутьТекст` | Текст пути без лишних пробелов. |
| `СоСлэшем` | Путь с завершающим `\` для склейки с именем файла. |

**Пример:** именованная область указывает на ячейку с `D:\Проект\` → запрос вернёт `D:\Проект\`.

### Запрос `Отбор_На_BASE`

| Имя | Смысл |
|-----|--------|
| `Папка` | Результат запроса пути. |
| `ИмяФайла` | Константа `Отбор.csv`. |
| `ПолныйПуть` | Полный путь к файлу. |
| `ДокументCsv` | Результат `Csv.Document`. |
| `ПерваяСтрокаЗаголовки` | Таблица с поднятыми заголовками. |

### Запрос `LOAD_Сводная`

| Имя | Смысл |
|-----|--------|
| `ТипыОтбора` | Список заголовков столбцов типов отбора (порядок фиксирован). |
| `fnСократитьФИО` | Локальная функция: полное ФИО → `Фамилия И.О.` |
| `fnВыбранДа` | Проверка, что `ВЫБРАН` равен 1 (число или текст). |
| `fnОтборСодержитТип` | Проверка вхождения подстроки типа в `ОТБОР` без учёта регистра. |
| `СтрокиДляАнализа` | Строки исходника с `ВЫБРАН = 1`. |
| `УникальныеПары` | Уникальные пары столбцов ТБ и ГОСБ по **всему** CSV. |
| `fnСобратьКмДляЯчейки` | Фильтрация и склейка уникальных сокращённых ФИО через `#(lf)`. |
| `СТиповымиКолонками` | Результат `List.Accumulate` — по одному столбцу на каждый тип отбора. |


## Замечания по данным

- В папке `IN/Отбор.csv` в репозитории при просмотре в UTF-8 заголовки могут отображаться некорректно: для Excel обычно используется **ANSI/Windows-1251**. Запрос настроен на **1251** и автоопределение разделителя (**`;`** или **`,`**).
- Если в вашем файле другая кодировка (например UTF-8 с BOM), замените `Encoding = 1251` на `Encoding = 65001` в `Отбор_На_BASE.pq`.
